Frequently Asked Questions

What can I do with the Hecatonchire project?

You can extend the memory accessible to your applications by utilizing the memory of physical servers on the network. In a similar way, You can extend the memory of Virtual Machines to create your own memory cloud for running unmodified high memory applications. You can use Hecatonchire technology to instantaneously live migrate virtual machine from one physical host to another regardless of the amount of memory the VM uses ( using post-copy migration )

What languages do you support?

The libheca API is written in C so supports C/C++. However, if you use VMs to create a scale-out memory cloud, any software can avail of the additional memory.


Do I need special Hardware?

No, you do not need any special hardware, the Hecatonchire project has been designed to run on off-the-shelf hardware. You do not need a special NIC to use the RDMA functionality. This is because we have included Softiwarp functionality by default in our modified kernel. You can also use SoftRoce if you prefer. (See Installing SoftIwarp and Installing SoftRoCE for details on how to use these.) That said, if you want to to truely benefit from hardware acceleration from RDMA enabled NICs, we support all RDMA fabrics that are OFED compatible (our testing has been against IB with Mellanox X2 and IWARP Chelsio T4 NIC).


Is it possible to use libheca without installing the heca kernel?

No, you will need to install the heca kernel.
