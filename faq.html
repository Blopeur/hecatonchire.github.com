Frequently Asked Questions

What can I do with Hecatonchire project

You can extend the available memory accessible by applications by tapping into the memory of physical servers on the network

You can extend the memory footprint of Virtual machine by create leveraging remote memory pool, creating a memory cloud

You can use Hecatonchire technology to instantaneously live migrate virtual machine from one physical host to another regardless of the amount of memory the VM use( post-copy method)

What languages do you support?

The libheca API is written in C so supports C/C++


Do I need special Hardware?

No you do not need any special hardware , Hecatonchire Is design to be run using of the shelves HW.

You do not need any special NIC to use the RDMA functionality as you can use Softiwarp as we included it by default in our modified kernel or you can use SoftRoce.

For more details on how to use softiwarp see Installing SoftIwarp or SoftRoCE Installing SoftRoCE.

However if you want to benefit from HW acceleration from RDMA enable NIC we support all type of fabric as long as they are OFED compatible ( We tested against IB with Mellanox X2 and IWARP Chelsio T4 NIC).


Is it possible to use libheca without installing the heca kernel?

No, you will need to install the heca kernel.
